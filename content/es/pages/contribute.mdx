**We welcome your contributions!**

We are a small all-volunteer, open-source project and the more support we have, the more impactful this project will be at serving communities on the front lines of resisting authoritarianism in the US.

## How you can support this project

<Button title="Contact us if you'd like to contribute →" href="/contact/" target="_self" size="lg" />

**Biggest needs**

* **Translation:** We will have an open-source translation system in place soon.
* **Writing:** New guides from people with real on-the-ground experience who can explain things plainly. See below.
* **Coding:** Check out [open issues on GitHub](https://github.com/ActivistChecklist/ActivistChecklist/issues).
* **Security review:** Help keep existing guides accurate as tools and threats change.
* **Promotion:** Help get this site into toolkits, networks, and communities that need it.

**Other needs**

* **Social media, copy editing, UX feedback, graphic design** — occasional or ongoing; reach out if it fits you.

***What about donations?*** Costs (hosting, translation, etc.) are real. [Contact us](/contact/) if you want to help fund work. (We don't have a donation system in place yet. Donations are not tax-deductible.)

## Contributing content and making edits

<Alert type="success">**It's easy to edit this site!** Anyone can contribute content to our site using our visual editor backend. All you need is a GitHub account. You don't have to know how to code or use GitHub.</Alert>

**1. Create a GitHub account** — if you don't have one already. Depending on your risk tolerance, you might consider creating an [anonymous GitHub account](https://drive.proton.me/urls/EPMFJ306KC#uM2jbPcD98xL) if you prefer to not have your identity tied to your contributions.

**2. Copy** the repository to your GitHub account.

<Button title="Copy the repository" href="https://github.com/ActivistChecklist/ActivistChecklist/fork" icon="IoOpenOutline" iconPosition="right" />

**3. Install the app** — connects your GitHub account to the visual editor. Give it permission just to access the Activist Checklist repository you just forked.

<Button title="Install Activist Checklist Keystatic app" href="https://github.com/apps/activistchecklist-keystatic/installations/new?state=close" icon="IoOpenOutline" iconPosition="right" />

**4. Open the visual editor**

<Button title="Open the visual editor" href="https://activistchecklist.up.railway.app/keystatic/" icon="IoOpenOutline" iconPosition="right" />

**5. Click "New branch..."** — use a short name that describes the changes you plan to make.

<Alert type="info">
  Note: You will get an error when you create the branch. Just refresh the page and you will see that it actually worked.
</Alert>

**6. Make your edits.** Find the guide or page you want to edit. If you want to edit a checklist item within a guide, open that guide first, then open the checklist item from there.

<ImageEmbed src="/images/content/contribute-keystatic-dashboard.png" alt="Visual editor dashboard: collections and entries" size="full" />

**7. Preview** (optional) — click the **Preview** button to see the site with your changes.

![Preview button in the visual editor](/images/content/contribute-preview-button.png)

**8. Click "Create pull request"** when you’re ready to submit your changes for review. We'll communicate on the pull request thread if we'd like to see any adjustments made before publishing.

![Create pull request control in GitHub](/images/content/contribute-pull-request.png)

## How we can collaborate on a new guide

You're welcome to write a guide from scratch using the process above. However, it can involve a lot of time and effort, and it's usually best if we collaborate so you create something we're likely to publish. Here's how that process can look:

* **Phase 1: Exploration.** [Reach out](/contact/) with the guide or article you have in mind. We’ll ask about your experience on the topic and for examples of your public writing that explains complex ideas in plain language.
* **Phase 2: Outline.** We send a template outline. We give feedback on the introduction and outline so you don’t invest in a full draft until the direction fits what we can publish.
* **Phase 3: Drafting.** We send a Proton Doc template that matches how content appears on the site (checklist items, “how to” and "alert" boxes, etc.). You submit a first draft. We arrange expert review and a few rounds of feedback. When it’s ready, we move to publishing.
* **Phase 4: Publishing.** We publish the piece on the site and share it through our usual channels. All guides are published anonymously for now.

## Process for contributing code

If you are a coder and want to help with the site, you can [fork our Github repo](https://github.com/ActivistChecklist/ActivistChecklist/fork) and submit a pull request with your changes.

Not sure where to start? Take a look at the [list of open issues](https://github.com/ActivistChecklist/ActivistChecklist/issues) and select one to work on. If the description of the issue is too brief or unclear, please ask for clarification on the issue thread.
